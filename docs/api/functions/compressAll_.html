      Batch stream variant of compressAll(). Emits per-file progress events with a fileIndex tag, then the final array of results.

      Memory warning: all files are held in memory. The maxConcurrent parameter (default 2) bounds parallel processing to prevent OOM on mobile.

      Parameters

      • files: (Blob | File)[]
      • options: CompressionOptions = {}
      • maxConcurrent: number = 2
      • svc: ImageCompression

      Returns AsyncIterable<CompressAllStreamEvent>

      Example

      for await (const evt of svc.compressAll$(files, { quality: 0.85 })) {
        if (isBatchResult(evt)) {
          // evt is CompressionResult[] — all done
          console.log(`Processed ${evt.length} files`);
        } else {
          // evt is BatchProgress
          console.log(`File ${evt.fileIndex}: ${evt.progress.percent}%`);
        }
      }

      Error handling: if any file fails, the entire batch throws (via the iterator's .throw()). For partial results, use compress() per file.
